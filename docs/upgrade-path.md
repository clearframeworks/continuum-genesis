# Upgrade Path

Continuum Genesis is MIT because the memory pattern should be visible, runnable, and useful.

## Genesis Layer

Use this when you want to:

- Try persistent memory locally
- Build a small prototype
- Inspect the shard format
- Connect an app to a simple memory runtime
- Learn the Continuum memory pattern

You are responsible for:

- Protecting the local data folder
- Reviewing shards before sending them to any model
- Avoiding sensitive customer or business data unless you add your own controls
- Backups, monitoring, access control, and deployment security

## Managed Continuum

Upgrade when you need:

- Security review before production use
- Hosted isolation
- A protected endpoint gated by Clear Frameworks access controls
- Stronger routing and memory selection
- Business workflow integration
- Customer portals, dashboards, reports, or automations
- Ongoing support and production operations

## Rule Of Thumb

If the memory is experimental, Genesis is a good starting point.

If the memory affects customers, operations, money, staff, compliance, or the reputation of a real business, use managed protocols and hosted features.
